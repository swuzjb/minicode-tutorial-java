Chapter 06

MCP 插件

工具写死在代码里。想接 GitHub API？改源码。想加数据库查询？改源码。MCP 解决这个问题。

到目前为止，MiniCode 的每一个工具都硬编码在 Java 源码中。想给它加一个"查 Jira 工单"的能力？你得写一个新的 Tool 实现类，重新编译，重新打包。这不是插件系统，这是改代码。

MCP（Model Context Protocol）解决了这个问题。它定义了一种标准协议，让 AI 宿主（我们的 MiniCode）可以与外部进程通信，动态发现和调用工具。就像 VS Code 的 Language Server Protocol，但面向 AI 工具调用。

MCP 的本质

MCP 本质上就是 JSON-RPC over stdin/stdout。宿主进程启动一个子进程，然后通过标准输入输出交换 JSON 消息。就这么简单。

一次完整的交互看起来像这样：

// Host → Server: 初始化
{"jsonrpc": "2.0", "id": 1, "method": "initialize", "params": {}}

// Host → Server: 发现工具
{"jsonrpc": "2.0", "id": 2, "method": "tools/list", "params": {}}

// Host → Server: 调用工具
{"jsonrpc": "2.0", "id": 3, "method": "tools/call",
 "params": {"name": "hello", "arguments": {"who": "world"}}}

Java MCP 客户端

用 ProcessBuilder 启动子进程，用 Jackson 序列化/反序列化 JSON-RPC 消息：

public class MCPClient {
    private final Process process;
    private final ObjectMapper mapper = new ObjectMapper();
    private int requestId = 0;
    private final BufferedReader reader;

    public MCPClient(String command, String... args) throws IOException {
        var pb = new ProcessBuilder();
        var cmd = new ArrayList<String>();
        cmd.add(command);
        cmd.addAll(List.of(args));
        pb.command(cmd);
        pb.redirectErrorStream(false);
        this.process = pb.start();
        this.reader = new BufferedReader(
            new InputStreamReader(process.getInputStream()));
    }

    public JsonNode sendRequest(String method, JsonNode params) throws IOException {
        var request = mapper.createObjectNode()
            .put("jsonrpc", "2.0")
            .put("id", ++requestId)
            .put("method", method);
        request.set("params", params != null ? params : mapper.createObjectNode());

        var line = mapper.writeValueAsString(request) + "\n";
        process.getOutputStream().write(line.getBytes());
        process.getOutputStream().flush();

        var responseLine = reader.readLine();
        return mapper.readTree(responseLine);
    }

    public List<ToolDefinition> listTools() throws IOException {
        var response = sendRequest("tools/list", null);
        var tools = response.get("result").get("tools");
        var result = new ArrayList<ToolDefinition>();
        for (var t : tools) {
            result.add(new ToolDefinition(
                t.get("name").asText(),
                t.get("description").asText(),
                t.get("inputSchema")
            ));
        }
        return result;
    }

    public String callTool(String name, JsonNode args) throws IOException {
        var params = mapper.createObjectNode()
            .put("name", name);
        params.set("arguments", args);
        var response = sendRequest("tools/call", params);
        var content = response.get("result").get("content").get(0);
        return content.get("text").asText();
    }

    public void close() {
        process.destroyForcibly();
    }
}

这就是一个完整的 MCP 客户端了。100 行 Java 代码。ProcessBuilder 负责进程管理，Jackson 负责序列化，BufferedReader 按行读取响应。

从配置文件加载

MCP 服务器的配置放在项目根目录的 .minicode/mcp.json：

{
  "servers": {
    "github": {
      "command": "npx",
      "args": ["@modelcontextprotocol/server-github"]
    },
    "hello": {
      "command": "java",
      "args": ["-jar", "hello-server.jar"]
    }
  }
}

启动时读取配置，启动所有 MCP 服务器，把它们的工具注册到 ToolRegistry：

private static List<MCPClient> loadMCPServers(ToolRegistry registry) throws IOException {
    var configFile = Path.of(".minicode/mcp.json");
    if (!Files.exists(configFile)) return List.of();

    var mapper = new ObjectMapper();
    var config = mapper.readTree(configFile.toFile());
    var servers = config.get("servers");
    var clients = new ArrayList<MCPClient>();

    var fields = servers.fields();
    while (fields.hasNext()) {
        var entry = fields.next();
        var serverConfig = entry.getValue();
        var command = serverConfig.get("command").asText();
        var argsNode = serverConfig.get("args");
        var argsList = new String[argsNode.size()];
        for (int i = 0; i < argsNode.size(); i++)
            argsList[i] = argsNode.get(i).asText();

        var client = new MCPClient(command, argsList);
        client.sendRequest("initialize", null); // 握手

        // 发现工具并注册
        for (var tool : client.listTools()) {
            registry.registerMCP(tool, client);
        }
        clients.add(client);
        System.out.println("  [mcp] Loaded: " + entry.getKey());
    }
    return clients;
}

写一个最小 MCP Server

为了验证客户端能正常工作，我们用 Java 写一个最简单的 MCP 服务器——它只有一个 "hello" 工具：

public class HelloServer {
    public static void main(String[] args) throws Exception {
        var mapper = new ObjectMapper();
        var reader = new BufferedReader(new InputStreamReader(System.in));

        String line;
        while ((line = reader.readLine()) != null) {
            var request = mapper.readTree(line);
            var method = request.get("method").asText();
            var id = request.get("id");

            JsonNode result = switch (method) {
                case "initialize" -> mapper.createObjectNode()
                    .set("serverInfo",
                        mapper.createObjectNode().put("name", "hello-server"));

                case "tools/list" -> {
                    var tool = mapper.createObjectNode()
                        .put("name", "hello")
                        .put("description", "Say hello to someone");
                    var schema = mapper.createObjectNode()
                        .put("type", "object");
                    schema.set("properties", mapper.createObjectNode()
                        .set("who", mapper.createObjectNode()
                            .put("type", "string")));
                    tool.set("inputSchema", schema);
                    var res = mapper.createObjectNode();
                    res.set("tools", mapper.createArrayNode().add(tool));
                    yield res;
                }

                case "tools/call" -> {
                    var who = request.at("/params/arguments/who").asText("world");
                    var text = mapper.createObjectNode()
                        .put("type", "text")
                        .put("text", "Hello, " + who + "!");
                    var res = mapper.createObjectNode();
                    res.set("content", mapper.createArrayNode().add(text));
                    yield res;
                }

                default -> mapper.createObjectNode();
            };

            var response = mapper.createObjectNode();
            response.put("jsonrpc", "2.0");
            response.set("id", id);
            response.set("result", result);
            System.out.println(mapper.writeValueAsString(response));
            System.out.flush();
        }
    }
}

注意 Java 21 的 switch 表达式和 yield 关键字。这让消息分发逻辑清晰得多。

进程生命周期管理

MCP 服务器是子进程。如果 MiniCode 崩溃，这些子进程可能变成僵尸进程。用 shutdown hook 来保证清理：

private static final List<MCPClient> mcpClients = new ArrayList<>();

// 在 main() 中注册 shutdown hook
Runtime.getRuntime().addShutdownHook(new Thread(() -> {
    mcpClients.forEach(MCPClient::close);
    System.err.println("[mcp] All servers shut down.");
}));
为什么用 ProcessBuilder 而不是 SDK？

截至目前，Java 的 MCP SDK 还在早期阶段。直接用 ProcessBuilder + Jackson 实现 JSON-RPC 反而更简单、更透明。你完全理解每一行代码在做什么。等 SDK 成熟后再迁移也不迟——协议是标准的，换客户端实现不影响 MCP 服务器。
