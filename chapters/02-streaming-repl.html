Chapter 02

流式对话

REPL 循环 + 流式输出 —— 打字机效果，多轮记忆

上一章我们的程序只能说一句话就退出了。这一章我们给它加上交互循环（REPL）和流式输出——让它像真正的 AI 聊天工具一样，逐字输出回复，并记住上下文。

第一步：加入 REPL

先不管流式，我们先让程序能多轮对话。用 Scanner 读取用户输入，用一个 List<MessageParam> 保存对话历史：

package com.minicode;

import com.anthropic.client.AnthropicClient;
import com.anthropic.models.messages.*;

import java.util.ArrayList;
import java.util.List;
import java.util.Scanner;

public class Main {
    private static final List<MessageParam> messages = new ArrayList<>();

    public static void main(String[] args) {
        var client = AnthropicClient.builder()
            .apiKey(System.getenv("ANTHROPIC_API_KEY"))
            .build();

        var scanner = new Scanner(System.in);
        System.out.println("MiniCode v0.1 (type \"exit\" to quit)\n");

        while (true) {
            System.out.print("you> ");
            var input = scanner.nextLine().trim();
            if (input.equals("exit")) break;
            if (input.isEmpty()) continue;

            messages.add(MessageParam.ofUser(input));

            var response = client.messages().create(
                MessageCreateParams.builder()
                    .model("claude-sonnet-4-20250514")
                    .maxTokens(2048)
                    .messages(messages)
                    .build()
            );

            var text = new StringBuilder();
            for (var block : response.content()) {
                if (block instanceof ContentBlock.Text t) {
                    text.append(t.text());
                }
            }
            System.out.println("\nassistant> " + text + "\n");

            messages.add(MessageParam.ofAssistant(response.content()));
        }
    }
}

关键点：每次 API 调用我们都传入完整的 messages 列表。Claude 自身是无状态的——它靠我们传递的消息历史来"记住"对话。

第二步：流式输出

上面的版本有个问题：你得等 Claude 把所有文字生成完才能看到结果。对于长回复，用户会盯着空白屏幕等好几秒。我们需要流式输出——像打字机一样逐字显示。

Anthropic Java SDK 提供了 createStreaming 方法。替换 API 调用部分：

var stream = client.messages().createStreaming(
    MessageCreateParams.builder()
        .model("claude-sonnet-4-20250514")
        .maxTokens(2048)
        .messages(messages)
        .build()
);

System.out.print("\nassistant> ");
var finalMessage = stream
    .onText(text -> System.out.print(text))
    .getFinalMessage();
System.out.println("\n");

messages.add(MessageParam.ofAssistant(finalMessage.content()));
流式 API 的工作原理

底层使用 Server-Sent Events (SSE)。服务器不再一次性返回完整 JSON，而是逐块推送 content_block_delta 事件，每个事件包含一小段文本。SDK 的 onText 回调在每收到一段文本时触发，getFinalMessage() 在流结束后返回组装好的完整消息对象。

完整的流式 REPL

把两步合在一起，这就是完整的 Main.java：

package com.minicode;

import com.anthropic.client.AnthropicClient;
import com.anthropic.models.messages.*;

import java.util.ArrayList;
import java.util.List;
import java.util.Scanner;

public class Main {
    private static final List<MessageParam> messages = new ArrayList<>();

    public static void main(String[] args) {
        var client = AnthropicClient.builder()
            .apiKey(System.getenv("ANTHROPIC_API_KEY"))
            .build();

        var scanner = new Scanner(System.in);
        System.out.println("MiniCode v0.1 (type \"exit\" to quit)\n");

        while (true) {
            System.out.print("you> ");
            var input = scanner.nextLine().trim();
            if (input.equals("exit")) break;
            if (input.isEmpty()) continue;

            messages.add(MessageParam.ofUser(input));

            var stream = client.messages().createStreaming(
                MessageCreateParams.builder()
                    .model("claude-sonnet-4-20250514")
                    .maxTokens(2048)
                    .messages(messages)
                    .build()
            );

            System.out.print("\nassistant> ");
            var finalMessage = stream
                .onText(text -> System.out.print(text))
                .getFinalMessage();
            System.out.println("\n");

            messages.add(MessageParam.ofAssistant(finalMessage.content()));
        }
    }
}

流式 vs 非流式

两种模式的对比：

特性 非流式 create() 流式 createStreaming()
首字节延迟 等全部生成完 约 200-500ms
用户体验 一次性显示全部 逐字打字机效果
超时风险 长回复可能超时 持续有数据不会超时
API 费用 相同 相同
关于 SSE

底层协议是 HTTP 的 Server-Sent Events。服务器端保持连接不关闭，用 data: 前缀逐行推送 JSON 事件。SDK 帮我们处理了所有协议细节——你只需要注册 onText 回调即可。
