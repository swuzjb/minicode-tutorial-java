Chapter 01

Hello, Claude

一个 main 方法，调通 API，拿到回复

我们从最简单的事情开始：写一个 main 方法，调用 Claude API，拿到回复，打印出来。整个过程不到 20 行 Java 代码。

创建 Maven 项目

打开终端，用 Maven archetype 生成项目骨架：

mvn archetype:generate \
  -DgroupId=com.minicode \
  -DartifactId=minicode \
  -DarchetypeArtifactId=maven-archetype-quickstart \
  -DinteractiveMode=false

进入项目目录后，编辑 pom.xml，设置 Java 21 并添加 Anthropic SDK 依赖：

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
         http://maven.apache.org/xsd/maven-4.0.0.xsd">
  <modelVersion>4.0.0</modelVersion>

  <groupId>com.minicode</groupId>
  <artifactId>minicode</artifactId>
  <version>0.1.0</version>

  <properties>
    <maven.compiler.source>21</maven.compiler.source>
    <maven.compiler.target>21</maven.compiler.target>
    <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
  </properties>

  <dependencies>
    <dependency>
      <groupId>com.anthropic</groupId>
      <artifactId>anthropic-java</artifactId>
      <version>1.1.0</version>
    </dependency>
  </dependencies>
</project>

项目结构此时长这样：

minicode/ ├── pom.xml └── src/main/java/com/minicode/ └── Main.java

设置 API Key

注意

永远不要把 API Key 提交到版本控制。后续章节会用配置文件管理，现在先用环境变量。

export ANTHROPIC_API_KEY="your-key-here"

Anthropic Java SDK 会自动从 ANTHROPIC_API_KEY 环境变量读取密钥。你也可以在代码里显式设置，但用环境变量更安全。

写代码

创建 src/main/java/com/minicode/Main.java——不到 20 行有效代码：

package com.minicode;

import com.anthropic.client.AnthropicClient;
import com.anthropic.models.messages.*;

public class Main {
    public static void main(String[] args) {
        var client = AnthropicClient.builder()
            .apiKey(System.getenv("ANTHROPIC_API_KEY"))
            .build();

        var message = client.messages().create(
            MessageCreateParams.builder()
                .model("claude-sonnet-4-20250514")
                .maxTokens(1024)
                .addUserMessage("用一句话解释什么是 Java")
                .build()
        );

        for (var block : message.content()) {
            if (block instanceof ContentBlock.Text text) {
                System.out.println(text.text());
            }
        }
    }
}

运行它

minicode — zsh
$ mvn compile exec:java -Dexec.mainClass="com.minicode.Main" -q
Java 是一种跨平台的面向对象编程语言，通过 JVM 实现"一次编写，到处运行"的理念。

就这样。一个 main 方法，你的 Java 程序已经在跟 Claude 对话了。

提示

如果你不想每次看到 Maven 的下载日志，加 -q（quiet 模式）。也可以在 pom.xml 里配置 exec-maven-plugin 让运行命令更短。

理解返回值

你可能注意到了——我们用 for 循环遍历 message.content()，并做了 instanceof 模式匹配，而不是直接取一个字符串。这是因为 Claude 的响应是一个内容块列表。它可以包含多种类型的块：

  • ContentBlock.Text — 普通文本回复
  • ContentBlock.ToolUse — 工具调用请求（第三章会用到）

Java 21 的 pattern matching for instanceof 让这个检查非常简洁——直接在类型判断的同时声明变量。

完整的响应对象在 JSON 层面长这样：

{
  "id": "msg_01XFDUDYJgAACzvnptvVoYEL",
  "type": "message",
  "role": "assistant",
  "content": [
    { "type": "text", "text": "Java 是..." }
  ],
  "stop_reason": "end_turn",
  "usage": {
    "input_tokens": 18,
    "output_tokens": 45
  }
}

在 Java SDK 中，你可以通过 message.usage() 获取 token 用量：

var usage = message.usage();
System.out.printf("tokens: %d in, %d out%n",
    usage.inputTokens(), usage.outputTokens());
// tokens: 18 in, 45 out

后续章节我们会用它来做 token 计数显示和上下文窗口管理。
