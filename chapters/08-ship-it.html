Chapter 08

打包发布

一直用 mvn exec:java 运行。是时候变成一个真正的命令行工具了。

到目前为止你一直在 IDE 里或者用 mvn exec:java -Dexec.mainClass="com.minicode.Main" 运行 MiniCode。想给同事用？"先装 Maven，然后 clone 仓库，然后..."——没人会用的。

这一章把 MiniCode 变成一个可以双击运行的 JAR，甚至一个原生可执行文件。

项目结构整理

经过前七章的迭代，最终的项目结构应该是这样的：

minicode/ ├── src/ │ ├── main/java/com/minicode/ │ │ ├── Main.java // 入口 + Agent Loop │ │ ├── ToolRegistry.java // 工具注册表 │ │ ├── Security.java // 权限规则引擎 │ │ ├── MCPClient.java // MCP 客户端 │ │ ├── ContextManager.java // Context 压缩 │ │ └── tools/ │ │ ├── ReadFileTool.java │ │ ├── WriteFileTool.java │ │ ├── BashTool.java │ │ └── GrepTool.java │ └── test/java/com/minicode/ │ ├── ToolRegistryTest.java │ └── SecurityTest.java ├── pom.xml └── minicode.sh

测试

发布之前先确保核心逻辑是对的。用 JUnit 5 写几个关键测试：

ToolRegistry 测试

import static org.junit.jupiter.api.Assertions.*;

class ToolRegistryTest {

    @Test
    void registryExecutesRegisteredTool() {
        var registry = new ToolRegistry();
        registry.register(new Tool() {
            public String name() { return "echo"; }
            public String description() { return "Echoes input"; }
            public Map<String, Object> inputSchema() {
                return Map.of();
            }
            public String execute(JsonNode input) {
                return input.get("text").asText();
            }
        });

        var input = new ObjectMapper()
            .createObjectNode().put("text", "hello");
        assertEquals("hello", registry.execute("echo", input));
    }

    @Test
    void registryReturnsErrorForUnknown() {
        var registry = new ToolRegistry();
        var result = registry.execute(
            "nope", new ObjectMapper().createObjectNode());
        assertTrue(result.contains("Error"));
    }
}

Security 测试

class SecurityTest {

    @Test
    void denysDangerousCommands() {
        var input = new ObjectMapper()
            .createObjectNode().put("command", "rm -rf /");
        assertEquals(
            Permission.DENY,
            Security.checkPermission("bash", input)
        );
    }

    @Test
    void allowsReadOperations() {
        var input = new ObjectMapper()
            .createObjectNode().put("path", "foo.java");
        assertEquals(
            Permission.ALLOW,
            Security.checkPermission("read_file", input)
        );
    }

    @Test
    void allowsSafeCommands() {
        var input = new ObjectMapper()
            .createObjectNode().put("command", "mvn test");
        assertEquals(
            Permission.ALLOW,
            Security.checkPermission("bash", input)
        );
    }

    @Test
    void asksForUnknownTools() {
        var input = new ObjectMapper()
            .createObjectNode().put("content", "hello");
        assertEquals(
            Permission.ASK,
            Security.checkPermission("write_file", input)
        );
    }
}

Fat JAR 打包

用 maven-shade-plugin 把所有依赖打进一个 JAR。这样用户只需要一个文件就能运行：

<build>
  <plugins>
    <plugin>
      <groupId>org.apache.maven.plugins</groupId>
      <artifactId>maven-shade-plugin</artifactId>
      <version>3.5.0</version>
      <executions>
        <execution>
          <phase>package</phase>
          <goals><goal>shade</goal></goals>
          <configuration>
            <transformers>
              <transformer implementation="org.apache.maven.plugins.shade.resource.ManifestResourceTransformer">
                <mainClass>com.minicode.Main</mainClass>
              </transformer>
            </transformers>
          </configuration>
        </execution>
      </executions>
    </plugin>
  </plugins>
</build>

构建和运行

# 打包（跳过测试加速，正式发布时不要跳）
mvn clean package -DskipTests

# 运行
java -jar target/minicode-1.0-SNAPSHOT.jar

一个命令就能启动。但还可以更简洁。

Shell 包装脚本

创建 minicode.sh，让用户可以像原生命令一样使用：

#!/bin/bash
SCRIPT_DIR=$( cd -- "$( dirname -- "${BASH_SOURCE[0]}" )" &> /dev/null && pwd )
java -jar "$SCRIPT_DIR/minicode.jar" "$@"
# 安装
cp target/minicode-1.0-SNAPSHOT.jar ~/bin/minicode.jar
cp minicode.sh ~/bin/minicode
chmod +x ~/bin/minicode

# 使用
minicode

进阶：GraalVM 原生镜像

如果你追求极致的启动速度，GraalVM 的 native-image 可以把 JAR 编译成原生可执行文件：

# 安装 GraalVM（macOS）
sdk install java 21.0.2-graalce

# 编译为原生可执行文件
native-image -jar target/minicode-1.0-SNAPSHOT.jar minicode

# 毫秒级启动！
./minicode
GraalVM 的代价

native-image 需要处理反射配置（Jackson 大量使用反射）、编译时间长（几分钟）、调试困难。对于一个 CLI 工具来说，Java 21 的启动速度（~300ms）已经足够好了。GraalVM 是锦上添花，不是必需品。

继续探索

MiniCode 是一个教学项目，但它的架构是真实的。以下是一些值得继续探索的方向：

  • Spring Boot 集成 — 用 Spring DI 管理 Tool 注册，用 Spring Shell 增强 CLI 体验
  • Quarkus CLI — Quarkus 的 Picocli 扩展 + GraalVM 原生编译，毫秒级启动
  • 多 Agent 协作 — 多个 Claude 实例分工合作：一个规划，一个执行，一个 review
  • Web UI — Spring WebFlux + WebSocket，把 MiniCode 变成 Web 应用
  • Virtual Threads — Java 21 虚拟线程并行执行多个工具调用
  • 持久化会话 — 把对话历史存到文件，下次继续

你用 Java 实现了一个 AI 编程助手的全部核心组件。从第一个 API 调用到可发布的命令行工具，每一步都是你自己写的。没有框架魔法，没有黑盒抽象。

去用它写代码吧。
