Chapter 09

可扩展行为系统

MCP 让别人加工具。但有时候不是想加「能做什么」，而是想改变「怎么做」—— 比如让它先写测试，或者遇到 bug 用系统化调试。

到目前为止，MiniCode 的行为是固定的：系统提示词写死在代码里，用户只能通过对话来引导 AI。这一章我们让行为本身变成可插拔的——Slash Commands 提供本地快捷操作，Skills 注入行为指令，Custom Instructions 持久化用户偏好。

Part 1：Slash Commands

Slash Commands 是用户在 REPL 中直接执行的本地命令，不经过 AI。类似 Claude Code 中的 /help、/clear、/compact。

SlashCommand 接口

先定义抽象：

public interface SlashCommand {
    String name();
    String description();
    void execute(String[] args, AgentContext context);
}

三个方法，没有多余的东西。AgentContext 包含当前会话的所有状态——对话历史、活跃的 Skills、SkillLoader 引用等。

CommandRegistry

和 ToolRegistry 思路一致——注册 + 分发：

public class CommandRegistry {
    private final Map<String, SlashCommand> commands = new LinkedHashMap<>();

    public void register(SlashCommand cmd) {
        commands.put(cmd.name(), cmd);
    }

    public boolean isCommand(String input) {
        return input.startsWith("/");
    }

    public boolean execute(String input, AgentContext context) {
        var parts = input.substring(1).split("\\s+");
        var name = parts[0];
        var args = Arrays.copyOfRange(parts, 1, parts.length);

        var cmd = commands.get(name);
        if (cmd == null) {
            System.out.println("Unknown command: /" + name + ". Type /help for list.");
            return false;
        }
        cmd.execute(args, context);
        return true;
    }
}

内置命令

注册几个常用命令：/help、/clear、/compact、/model、/skill：

// 初始化时注册
var cmdRegistry = new CommandRegistry();

cmdRegistry.register(new SlashCommand() {
    public String name() { return "help"; }
    public String description() { return "Show available commands"; }
    public void execute(String[] args, AgentContext ctx) {
        cmdRegistry.getAll().forEach((n, c) ->
            System.out.printf("  /%s - %s%n", n, c.description()));
    }
});

cmdRegistry.register(new SlashCommand() {
    public String name() { return "clear"; }
    public String description() { return "Clear conversation history"; }
    public void execute(String[] args, AgentContext ctx) {
        ctx.messages().clear();
        System.out.println("Conversation cleared.");
    }
});

cmdRegistry.register(new SlashCommand() {
    public String name() { return "compact"; }
    public String description() { return "Compress conversation context"; }
    public void execute(String[] args, AgentContext ctx) {
        ctx.contextManager().compact(ctx.messages());
        System.out.println("Context compacted.");
    }
});

cmdRegistry.register(new SlashCommand() {
    public String name() { return "model"; }
    public String description() { return "Switch model (e.g. /model sonnet)"; }
    public void execute(String[] args, AgentContext ctx) {
        if (args.length == 0) {
            System.out.println("Current: " + ctx.model());
        } else {
            ctx.setModel(args[0]);
            System.out.println("Switched to: " + args[0]);
        }
    }
});

Part 2：Skill 加载系统

Slash Commands 是本地操作。Skill 不同——它改变 AI 的行为方式。一个 Skill 本质上是一段 Markdown 指令，被注入到 System Prompt 中，让 AI 在接下来的对话中遵循特定工作流。

比如加载一个 "tdd" Skill，AI 就会先写测试再实现；加载 "debugging" Skill，AI 会按系统化步骤排查问题。

Skill 数据模型

public record Skill(String name, String content, List<String> triggers) {
    public Skill(String name, String content) {
        this(name, content, List.of());
    }
}

用 Record 来表示——不可变，语义清晰。triggers 是可选的关键词列表，用于后面的自动建议功能。

SkillLoader

从文件系统发现和加载 Skill。约定目录结构：.minicode/skills/ 目录下，每个 .md 文件就是一个 Skill：

public class SkillLoader {
    private final Path skillsDir;
    private final Map<String, Skill> cache = new HashMap<>();

    public SkillLoader(Path skillsDir) {
        this.skillsDir = skillsDir;
    }

    public List<String> discover() throws IOException {
        if (!Files.exists(skillsDir)) return List.of();
        try (var stream = Files.list(skillsDir)) {
            return stream
                .filter(p -> p.toString().endsWith(".md"))
                .map(p -> p.getFileName().toString().replace(".md", ""))
                .toList();
        }
    }

    public Skill load(String name) throws IOException {
        if (cache.containsKey(name)) return cache.get(name);

        var path = skillsDir.resolve(name + ".md");
        var content = Files.readString(path);
        var skill = new Skill(name, content);
        cache.put(name, skill);
        return skill;
    }
}

注入 System Prompt

Skill 的核心机制：将加载的 Skill 内容拼接到 System Prompt 中。AI 只看到 System Prompt，它不知道这些指令来自 Skill 文件：

public String buildSystemPrompt(String base, List<Skill> activeSkills) {
    if (activeSkills.isEmpty()) return base;

    var sb = new StringBuilder(base);
    sb.append("\n\n## Active Skills\n\n");

    for (var skill : activeSkills) {
        sb.append("<skill name=\"").append(skill.name()).append("\">\n");
        sb.append(skill.content());
        sb.append("\n</skill>\n\n");
    }

    return sb.toString();
}

/skill 命令实现

将 Skill 管理暴露为 Slash Command：

public class SkillCommand implements SlashCommand {
    public String name() { return "skill"; }
    public String description() { return "Manage skills: list / load <name> / unload <name>"; }

    public void execute(String[] args, AgentContext ctx) {
        if (args.length == 0 || args[0].equals("list")) {
            try {
                var available = ctx.skillLoader().discover();
                var activeNames = ctx.activeSkills().stream()
                    .map(Skill::name).toList();
                for (var name : available) {
                    var marker = activeNames.contains(name) ? "●" : "○";
                    System.out.println("  " + marker + " " + name);
                }
            } catch (IOException e) {
                System.out.println("Error listing skills: " + e.getMessage());
            }
        } else if (args[0].equals("load") && args.length > 1) {
            try {
                var skill = ctx.skillLoader().load(args[1]);
                ctx.activeSkills().add(skill);
                System.out.println("Loaded skill: " + skill.name());
            } catch (IOException e) {
                System.out.println("Error: skill not found - " + args[1]);
            }
        } else if (args[0].equals("unload") && args.length > 1) {
            ctx.activeSkills().removeIf(s -> s.name().equals(args[1]));
            System.out.println("Unloaded skill: " + args[1]);
        }
    }
}

Part 3：Custom Instructions

Skill 需要手动加载。但有些偏好是持久的——比如"我总是用 JUnit 5"、"代码注释用中文"。Custom Instructions 让这些偏好自动生效。

项目级指令

从项目根目录读取配置文件。支持多种约定名称（兼容 Claude Code 的 CLAUDE.md）：

public String loadProjectInstructions() {
    var candidates = List.of(
        Path.of(".minicode", "instructions.md"),
        Path.of("AGENTS.md"),
        Path.of("CLAUDE.md")
    );

    for (var p : candidates) {
        var resolved = Path.of(System.getProperty("user.dir")).resolve(p);
        if (Files.exists(resolved)) {
            try { return Files.readString(resolved); }
            catch (IOException e) { /* continue */ }
        }
    }
    return "";
}

全局指令

用户级别的偏好放在 Home 目录。不管在哪个项目里都会生效：

public String loadGlobalInstructions() {
    var path = Path.of(System.getProperty("user.home"), ".minicode", "instructions.md");
    try { return Files.readString(path); }
    catch (IOException e) { return ""; }
}

分层 Prompt 组装

把所有来源的指令按优先级拼装成最终的 System Prompt：

public String assembleSystemPrompt(AgentContext ctx) {
    var layers = new ArrayList<String>();
    layers.add(ctx.baseSystemPrompt());

    var global = loadGlobalInstructions();
    if (!global.isBlank()) layers.add(global);

    var project = loadProjectInstructions();
    if (!project.isBlank()) layers.add(project);

    for (var skill : ctx.activeSkills()) {
        layers.add("<skill name=\"" + skill.name() + "\">\n" + skill.content() + "\n</skill>");
    }

    return String.join("\n\n---\n\n", layers);
}
优先级规则

优先级从低到高：基础指令 → 全局偏好 → 项目规则 → 主动加载的 Skills。和 Claude Code 的 CLAUDE.md 分层机制一致。后加载的内容优先级更高，可以覆盖前面的指令。

Part 4：自动技能建议（进阶）

更进一步：根据用户消息的关键词，主动建议加载对应的 Skill。不强制，只提示。

public Optional<Skill> suggestSkill(String message, List<Skill> available) {
    var lower = message.toLowerCase();
    if (lower.contains("bug") || lower.contains("error") || lower.contains("不工作")) {
        return available.stream()
            .filter(s -> s.name().equals("debugging"))
            .findFirst();
    }
    if (lower.contains("测试") || lower.contains("test")) {
        return available.stream()
            .filter(s -> s.name().equals("tdd"))
            .findFirst();
    }
    return Optional.empty();
}

在 REPL 中使用：

// 发送给 AI 前检查
var suggestion = suggestSkill(input, allSkills);
suggestion.ifPresent(skill -> {
    if (!ctx.activeSkills().contains(skill)) {
        System.out.printf("  [hint] Consider: /skill load %s%n", skill.name());
    }
});

本章小结

Slash Commands
本地快捷操作。不经过 AI，直接在 REPL 中执行。清除对话、切换模型、管理 Skills。
Skills
可加载的行为指令集。注入 System Prompt，改变 AI 的工作方式。TDD、调试、Code Review 都可以是 Skill。
Custom Instructions
持久化的偏好设定。全局级和项目级分层，自动加载，无需手动操作。

MCP 扩展「能做什么」，Skill 系统扩展「怎么做」。两者组合让 MiniCode 成为真正可定制的编程助手。

工具决定能力边界，行为决定工作质量。这是两个正交的扩展维度——一个向外扩展功能，一个向内提升方法。到这里，MiniCode 已经具备了一个生产级 AI 编程助手的全部核心架构。
