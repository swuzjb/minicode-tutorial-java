Chapter 04

工具注册表

消除重复，提取模式 —— 加入写文件、执行命令、搜索

上一章结束时，executeTool 方法里只有一个 if 分支。如果我们要加第二个、第三个工具，每次都要修改那个方法，加更多 else if——这很快会变得不可维护。

这一章我们做 Thorsten Ball 方法论里的关键一步：看到重复后提取抽象。我们定义一个 MinicodeTool 接口和一个 ToolRegistry，然后分别实现 read_file、write_file 和 bash 工具。

定义工具接口

每个工具需要提供：名字、描述、参数 Schema、执行逻辑。我们用一个 Java 接口表达：

package com.minicode.tools;

import com.fasterxml.jackson.databind.JsonNode;
import java.util.Map;

public interface MinicodeTool {
    String name();
    String description();
    Map<String, Object> inputSchema();
    String execute(JsonNode input) throws Exception;
}

简洁。每个工具只需要实现这四个方法。注意 execute 声明了 throws Exception——错误处理交给注册表统一做。

ToolRegistry

注册表负责存储工具、转换为 API 定义格式、路由执行：

package com.minicode.tools;

import com.anthropic.core.JsonValue;
import com.anthropic.models.messages.Tool;
import com.fasterxml.jackson.databind.JsonNode;

import java.util.HashMap;
import java.util.List;
import java.util.Map;

public class ToolRegistry {
    private final Map<String, MinicodeTool> tools = new HashMap<>();

    public void register(MinicodeTool tool) {
        tools.put(tool.name(), tool);
    }

    /** 转为 Anthropic API 需要的 Tool 定义列表 */
    public List<Tool> definitions() {
        return tools.values().stream().map(t ->
            Tool.builder()
                .name(t.name())
                .description(t.description())
                .inputSchema(buildSchema(t.inputSchema()))
                .build()
        ).toList();
    }

    /** 根据名字路由执行，统一捕获异常 */
    public String execute(String name, JsonNode input) {
        var tool = tools.get(name);
        if (tool == null) {
            return "Error: unknown tool \"" + name + "\"";
        }
        try {
            return tool.execute(input);
        } catch (Exception e) {
            return "Error: " + e.getMessage();
        }
    }

    private Tool.InputSchema buildSchema(Map<String, Object> schema) {
        var builder = Tool.InputSchema.builder();
        for (var entry : schema.entrySet()) {
            builder.putAdditionalProperty(
                entry.getKey(), JsonValue.from(entry.getValue())
            );
        }
        return builder.build();
    }
}

ReadFileTool

重构上一章的读文件逻辑，加上行号显示——这样 Claude 引用代码时能给出精确位置：

package com.minicode.tools;

import com.fasterxml.jackson.databind.JsonNode;
import java.nio.file.*;
import java.util.*;

public class ReadFileTool implements MinicodeTool {

    public String name() { return "read_file"; }

    public String description() {
        return "Read the contents of a file with line numbers";
    }

    public Map<String, Object> inputSchema() {
        return Map.of(
            "type", "object",
            "properties", Map.of(
                "path", Map.of("type", "string", "description", "File path to read")
            ),
            "required", List.of("path")
        );
    }

    public String execute(JsonNode input) throws Exception {
        var path = input.get("path").asText();
        var lines = Files.readAllLines(Path.of(path));
        var sb = new StringBuilder();
        for (int i = 0; i < lines.size(); i++) {
            sb.append(i + 1).append(": ").append(lines.get(i)).append("\n");
        }
        return sb.toString();
    }
}

WriteFileTool

让 Claude 能写文件——这是编程助手最关键的能力之一：

package com.minicode.tools;

import com.fasterxml.jackson.databind.JsonNode;
import java.nio.file.*;
import java.util.*;

public class WriteFileTool implements MinicodeTool {

    public String name() { return "write_file"; }

    public String description() {
        return "Write content to a file, creating parent directories if needed";
    }

    public Map<String, Object> inputSchema() {
        return Map.of(
            "type", "object",
            "properties", Map.of(
                "path", Map.of("type", "string", "description", "File path to write"),
                "content", Map.of("type", "string", "description", "Content to write")
            ),
            "required", List.of("path", "content")
        );
    }

    public String execute(JsonNode input) throws Exception {
        var path = Path.of(input.get("path").asText());
        var content = input.get("content").asText();

        // 自动创建父目录
        if (path.getParent() != null) {
            Files.createDirectories(path.getParent());
        }
        Files.writeString(path, content);
        return "Wrote " + content.length() + " chars to " + path;
    }
}

BashTool

这是最强大也最危险的工具——执行任意 Shell 命令：

package com.minicode.tools;

import com.fasterxml.jackson.databind.JsonNode;
import java.io.File;
import java.util.*;
import java.util.concurrent.TimeUnit;

public class BashTool implements MinicodeTool {

    public String name() { return "bash"; }

    public String description() {
        return "Execute a shell command and return its output";
    }

    public Map<String, Object> inputSchema() {
        return Map.of(
            "type", "object",
            "properties", Map.of(
                "command", Map.of("type", "string", "description", "Shell command to execute")
            ),
            "required", List.of("command")
        );
    }

    public String execute(JsonNode input) throws Exception {
        var command = input.get("command").asText();

        var process = new ProcessBuilder("sh", "-c", command)
            .directory(new File(System.getProperty("user.dir")))
            .redirectErrorStream(true)
            .start();

        var output = new String(process.getInputStream().readAllBytes());
        boolean finished = process.waitFor(30, TimeUnit.SECONDS);

        if (!finished) {
            process.destroyForcibly();
            return "Error: command timed out after 30s";
        }

        return output.isEmpty() ? "(no output)" : output;
    }
}
为什么用 ProcessBuilder？

用 ProcessBuilder 而不是 Runtime.exec()。ProcessBuilder 提供更好的进程控制：可以设置工作目录、合并 stdout/stderr 流（redirectErrorStream）、设置超时。Runtime.exec() 在参数解析和流处理上有众所周知的陷阱。

组装

在 Main.java 里创建注册表并注册工具：

var registry = new ToolRegistry();
registry.register(new ReadFileTool());
registry.register(new WriteFileTool());
registry.register(new BashTool());

// API 调用时传入 registry.definitions()
var response = client.messages().create(
    MessageCreateParams.builder()
        .model("claude-sonnet-4-20250514")
        .maxTokens(4096)
        .tools(registry.definitions())  // 自动转换格式
        .messages(messages)
        .build()
);

// 执行工具时通过 registry 路由
var result = registry.execute(toolUse.name(), toolUse.input());

现在新增一个工具只需要两步：写一个实现 MinicodeTool 的类，然后 registry.register() 注册。Agent Loop 本身不需要任何修改。

设计回顾

注册表模式：Agent Loop 不知道也不关心具体有哪些工具

为什么不用注解或反射？

你可能想用 @Tool 注解 + 类路径扫描来自动发现工具。先别急——Thorsten Ball 的方法论是"在需要之前不要抽象"。三个工具手动注册完全清晰。如果未来工具数量超过十个，再考虑自动发现。

安全隐患

现在 BashTool 可以执行任何命令——包括 rm -rf /。WriteFileTool 也可以覆盖系统文件。这些工具在没有任何防护的情况下直接暴露给 AI 是非常危险的。下一章我们加上安全防护。
